# How to Create a Pull Request

Follow these steps to submit your changes for review:

1. **Fork the repository** (if you donâ€™t have write access).
2. **Create a new branch** for your feature or fix, following the branch naming conventions in CONTRIBUTING.md.
3. **Commit your changes** using the commit message format described in CONTRIBUTING.md.
4. **Push your branch** to the remote repository.
5. **Open a Pull Request**:
   - Go to the repository on GitHub.
   - Click "Compare & pull request" next to your branch.
   - Fill in the PR title and description:
     - Clearly describe what your changes do.
     - Reference related issues if applicable.
     - Add a TL;DR summary at the end.
6. **Request review** from maintainers or relevant team members.
7. **Respond to feedback** and make any required changes.
8. **Wait for approval and merge** (maintainers will handle merging).

See `docs/pull_request_message.md` for a template.

## Tips
- Keep your PR focused and concise.
- Make sure your branch is up to date with the main branch before opening the PR.
- Review your own code before submitting.
